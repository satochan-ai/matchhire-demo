# MatchHire

採用プロセスをひとつの画面で管理・分析できるツールです。
スカウト送信から内定・入社までの流れを可視化し、どこで候補者が離脱しているかをすぐに把握できます。

---

## できること

### 📊 採用ファネルの分析（ダッシュボード）
スカウト送信 → 開封 → 返信 → 有効応募 → 面接 → 内定 の各ステップを一画面で確認できます。
どのステップで数が落ちているか（ボトルネック）を自動で診断し、改善アクションを提案します。

### 👤 候補者管理
候補者の一覧表示・絞り込み・詳細確認ができます。
各候補者の選考ステータス、面接履歴、接触履歴をまとめて見られます。

### 📋 応募管理
どの候補者がどの求人に応募しているかを一覧で確認できます。
有効応募・無効応募のフィルタリングにも対応しています。

### 💼 求人管理
求人ごとの応募数・有効応募数・面接数・内定数を比較できます。
求人の詳細ページでは、その求人への応募者一覧も確認できます。

### 🧑‍💼 担当者分析
採用担当者ごとのスカウト送信数・返信率・有効応募数を比較できます。
誰のスカウトが効果的かをデータで確認できます。

### ❌ NG理由分析
不採用・辞退になった理由の内訳を円グラフで確認できます。
チャネルや担当者でフィルタリングして深掘りできます。

---

## デモの見方

1. **ダッシュボード**（左メニュー最上部）から全体の採用状況を確認します
2. 右上の **期間フィルター** で「今月」「先月」「過去3ヶ月」などに切り替えられます
3. ボトルネック診断のカードで、現在どこが課題かが一目でわかります
4. 左メニューから各ページ（候補者・応募・求人・担当者・NG理由）に移動できます
5. テーブルの候補者名・求人名はクリックすると詳細ページに遷移します

## 起動方法

### 必要なもの
- [Node.js](https://nodejs.org/) バージョン 18 以上

### 手順

```bash
# 1. リポジトリをクローン（または ZIP ダウンロード後に展開）
git clone <リポジトリURL>
cd match-hire

# 2. 依存パッケージをインストール（初回のみ）
npm install

# 3. 開発サーバーを起動
npm run dev
```

起動後、ブラウザで [http://localhost:3000](http://localhost:3000) を開いてください。
自動的にダッシュボードページへ移動します。

## データソースについて

現在はすべて **モックデータ（ダミーデータ）固定** で動作しています。
`.env.local` ファイルの設定は以下のとおりです：

```
NEXT_PUBLIC_DATA_SOURCE=mock
```

> 将来的に Google スプレッドシートや CSV ファイルへの切り替えにも対応する設計になっています。
> 切り替え方法は `docs/csv_operation_guide.md` を参照してください。

---

## 注意事項

- 画面上に表示されている人名・スキル・会社情報などは **すべて架空のダミーデータ** です
- 実在する人物・企業・求人とは一切関係ありません
- 個人情報は一切含まれていません。安全にデモ・開発にご利用いただけます

## ディレクトリ構成（概要）

```

├── src/
│   ├── app/            # 各ページのファイル
│   ├── components/     # 画面部品（ボタン・テーブルなど）
│   ├── hooks/          # データ取得ロジック
│   └── lib/            # 計算・データ処理ロジック

└── .env.local          # 環境変数（データソース設定）
```
