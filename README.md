# MatchHire

🔗 **デモ（GitHub Pages）**: https://satochan-ai.github.io/matchhire-demo/

> デモ版はダミーデータ（mock固定）で動作します。個人情報・実データは含まれません。

**採用の「なんとなく」をデータに変えるツールです。**

スカウト送信から内定・入社まで、採用活動のすべてをひとつの画面で管理・分析できます。
「どこで候補者が離脱しているか」「誰のスカウトが効いているか」が、すぐにわかります。

---

## こんな課題を解決します

| よくある悩み | MatchHire でできること |
|---|---|
| 候補者の進捗をExcelで管理していて更新が大変 | 候補者・応募・面接履歴を一元管理 |
| スカウトをたくさん送っているのに面接につながらない | ファネルのどこが詰まっているか自動で診断 |
| 担当者によって採用結果がバラバラ | 担当者ごとの返信率・通過率を数字で比較 |
| なぜ不採用・辞退になったのか分析できていない | NG理由を集計してグラフで可視化 |
| 採用状況を上司や経営層に報告するのが手間 | ダッシュボードをそのまま見せるだけでOK |

---

## 主な機能

### 📊 ダッシュボード（採用ファネル全体を一画面で）

```
スカウト送信 → 開封 → 返信 → 有効応募 → 面接 → 内定 → 承諾 → 入社
```

各ステップの件数と転換率を一覧表示。
「返信は来ているのに有効応募にならない」「面接数は多いのに内定が出ない」
といった課題が、グラフを見るだけで把握できます。

**ボトルネック診断機能**では、現在の最大課題を自動で特定し、
改善のための具体的なアクション（例：「スカウト文面の冒頭を候補者のキャリアへの共感から始める」）
まで提示します。

---

### 👤 候補者管理（進捗を一覧で把握）

- 全候補者の名前・スキル・選考ステータスを一覧表示
- 「書類選考中」「面接中」「内定」などのステータスで絞り込み可能
- 候補者名をクリックすると詳細ページへ遷移
  - 基本情報・選考状況・面接履歴・接触履歴がすべて1ページにまとまっています

---

### 📋 応募管理（どの求人に誰が応募しているかを把握）

- 全応募を一覧表示（求人名・応募日・応募経路・有効応募フラグ）
- 「有効応募のみ表示」「スカウト経由のみ」などのフィルターで絞り込み可能

---

### 💼 求人管理（求人ごとの採用状況を比較）

- 求人ごとの応募数・有効応募数・面接数・内定数を一覧で比較
- 求人名をクリックすると、その求人への応募者一覧・現在の選考ステータスを確認できます

---

### 🧑‍💼 担当者分析（誰が成果を出しているかを可視化）

- 採用担当者ごとのスカウト送信数・返信率・有効応募数・面接数を比較
- 「Aさんは送信数は多いが返信率が低い」「Bさんは少数精鋭で内定率が高い」といった差が一目でわかります

---

### ❌ NG理由分析（なぜ落ちているのかを集計）

- 不採用・辞退の理由を円グラフで可視化
- チャネル・担当者・期間でフィルタリングして深掘りできます
- 「スキルアンマッチ」が多い → 求人要件の見直しが必要、といった判断に役立てられます

---

## 画面の使い方

起動後、ブラウザで以下のURLにアクセスしてください。
左側のメニューからも各ページに移動できます。

| ページ | URL | 見どころ |
|---|---|---|
| ダッシュボード | `/matchhire/dashboard` | ファネル全体・ボトルネック・AIインサイト |
| 候補者管理 | `/matchhire/candidates` | 候補者一覧・ステータス絞り込み |
| 候補者詳細 | `/matchhire/candidates/1` | 面接履歴・接触履歴・選考状況 |
| 応募管理 | `/matchhire/applications` | 応募一覧・有効応募フィルタ |
| 求人管理 | `/matchhire/jobs` | 求人ごとの採用数比較 |
| 求人詳細 | `/matchhire/jobs/j1` | 応募者一覧・現在ステータス |
| 担当者分析 | `/matchhire/owners` | 担当者別パフォーマンス |
| NG理由分析 | `/matchhire/ng-reasons` | 不採用・辞退理由の内訳 |

### はじめて触るときの順番

1. **ダッシュボード** を開き、右上の期間フィルターで「今月」を選択
2. ファネルグラフでどのステップが細くなっているか確認
3. ボトルネック診断カードで課題と改善アクションを確認
4. **担当者分析** で担当者ごとの数字を比較
5. **候補者管理** で候補者名をクリック → 詳細ページで履歴を確認

## 起動方法

**事前に必要なもの：** [Node.js](https://nodejs.org/ja/) バージョン 18 以上

```bash
# ① リポジトリをダウンロード
git clone <リポジトリURL>
cd match-hire

# ② 必要なファイルをインストール（初回のみ・1〜2分かかります）
npm install

# ③ 起動
npm run dev
```

起動後、ブラウザで **[http://localhost:3000](http://localhost:3000)** を開くとダッシュボードが表示されます。

## データモード

MatchHire は、データの取得元を3つのモードから選べる設計になっています。
設定は `.env.local` というファイルの1行を書き換えるだけです。

| モード | 設定値 | 内容 | 用途 |
|---|---|---|---|
| **モック**（初期設定） | `mock` | ツール内蔵のダミーデータ | デモ・動作確認 |
| **CSV** | `csv` | `data/` フォルダのCSVファイル | 自社データで試したいとき |
| **Google Sheets** | `sheets` | Googleスプレッドシートと連携 | 本番運用（準備中） |

```
# .env.local（プロジェクトのルートフォルダにあります）
NEXT_PUBLIC_DATA_SOURCE=mock
```

> **CSVモードの使い方**は [`docs/csv_operation_guide.md`](docs/csv_operation_guide.md) に詳しく書いてあります。
> Excelで作ったデータをCSV形式で書き出して `data/` フォルダに置くだけで、自社のデータで動かせます。

## GitHub Pages デモについて

| 項目 | 内容 |
|---|---|
| デモURL | https://satochan-ai.github.io/matchhire-demo/ |
| データ | mockData 固定（ダミーデータ） |
| API Route | 使用しない（静的エクスポート） |

### ローカルで静的ビルドを確認したい場合

```bash
# macOS / Linux
GITHUB_PAGES=true NEXT_PUBLIC_DATA_SOURCE=mock npm run build:gh

# Windows (PowerShell)
$env:GITHUB_PAGES="true"; $env:NEXT_PUBLIC_DATA_SOURCE="mock"; npm run build:gh
```

`out/` フォルダが生成されれば成功です。

### ディレクトリ構成

```
match-hire/
├── src/
│   ├── app/            # 各ページ（Next.js App Router）
│   ├── components/     # UIコンポーネント
│   ├── hooks/          # データ取得フック（useRawData）
│   └── lib/            # データ処理・計算ロジック / リポジトリ層
├── data/               # CSVファイル置き場（CSVモード時に使用）
├── docs/               # 運用ガイド・設計ドキュメント
├── gas/                # Google Apps Script（Sheetsモード用）
└── .env.local          # データソース設定
```

### データソース切り替え

`src/lib/dataSource.ts` の `getDataSource()` が `NEXT_PUBLIC_DATA_SOURCE` を読み取り、
各リポジトリ（`src/lib/repositories/`）がモード別の取得ロジックに分岐します。
