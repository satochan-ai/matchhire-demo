# bottleneck_logic.md

## MatchHire：ボトルネック診断ロジック

---

## 1. 概要

本ロジックは、採用ファネルにおける各KPIを基に
**どこで詰まっているか（ボトルネック）を自動判定する仕組み**である。

---

## 2. 判定対象KPI

* 開封率（open_rate）
* 返信率（reply_rate）
* 有効応募率（valid_rate）
* 書類通過率（document_pass_rate）
* 面接通過率（interview_pass_rate）
* 内定率（offer_rate）
* 承諾率（accept_rate）

---

## 3. 閾値定義（初期値）

| KPI   | Good | Warning | Alert |
| ----- | ---- | ------- | ----- |
| 開封率   | ≥60% | 40〜59%  | <40%  |
| 返信率   | ≥30% | 15〜29%  | <15%  |
| 有効応募率 | ≥70% | 50〜69%  | <50%  |
| 書類通過率 | ≥50% | 30〜49%  | <30%  |
| 面接通過率 | ≥50% | 30〜49%  | <30%  |
| 内定率   | ≥30% | 15〜29%  | <15%  |
| 承諾率   | ≥60% | 40〜59%  | <40%  |

---

## 4. ボトルネック判定ロジック

### 4.1 基本ルール

* Alertが最優先
* 複数ある場合は上流優先
* 最初に詰まっている箇所を特定

---

### 4.2 判定フロー

```id="logic123"
if open_rate < 40% → 上流（開封）
else if reply_rate < 15% → 上流（返信）
else if valid_rate < 50% → 中流（応募質）
else if document_pass_rate < 30% → 下流（書類）
else if interview_pass_rate < 30% → 下流（面接）
else if offer_rate < 15% → 下流（内定）
else if accept_rate < 40% → 下流（承諾）
else → 問題なし
```

---

## 5. 診断コメント生成

### 上流（開封率低）

* 「スカウトタイトル・対象選定に問題があります」

### 上流（返信率低）

* 「文面の魅力・訴求不足の可能性があります」

### 中流（有効応募率低）

* 「ターゲット選定がズレています」

### 下流（書類通過率低）

* 「要件と候補者のミスマッチが発生しています」

### 下流（面接通過率低）

* 「面接評価または候補者の質に課題があります」

### 下流（承諾率低）

* 「条件・魅力・クロージングに課題があります」

---

## 6. 優先度判定

* Alert → 最優先改善
* Warning → 改善検討
* Good → 維持

---

## 7. 出力形式（UI用）

```json id="json1"
{
  "status": "Alert",
  "stage": "返信率",
  "message": "文面の魅力が不足しています",
  "suggestion": "テンプレートの見直しを推奨"
}
```

---

## 8. 特徴

* 営業分析ロジックを採用に応用
* 上流から順に評価
* 改善アクションまで提示
