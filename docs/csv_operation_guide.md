# CSV 運用ガイド

> MatchHire — データ更新・運用手順書  
> 対象者：採用担当者・人事部スタッフ（非エンジニア向け）  
> 作成日：2026-04-26

---

## はじめに

MatchHire の「CSVモード」では、Google スプレッドシートで管理している採用データを CSV ファイルに書き出してアプリに読み込ませます。  
データを更新するたびにこのガイドの手順を実施してください。

**全体の流れ（5ステップ）**

```
① スプレッドシートを更新
  ↓
② CSV ファイルとして書き出す
  ↓
③ data フォルダに上書き配置
  ↓
④ アプリを再起動
  ↓
⑤ ブラウザで確認
```

---

## 1. CSV 更新ルール

データを更新する前に必ず以下のルールを確認してください。

### 1-1. 更新できるもの（OK）

| 操作 | 具体例 |
|---|---|
| 行の追加 | 新しい候補者・求人・応募を追加する |
| 既存行の値の変更 | 候補者のステータスを「書類選考」→「面接」に変更する |
| 行の削除 | 取り消しになった応募を削除する |

### 1-2. 絶対にやってはいけないこと（NG）

| 操作 | 理由 |
|---|---|
| **1行目（ヘッダー行）のカラム名を変更する** | アプリがデータを読み込めなくなります |
| **カラムの順番を入れ替える** | データが別のカラムに入り表示が崩れます |
| **カラムを削除する** | 必須項目が欠落し、アプリがエラーになります |
| **カラムを追加する** | 現時点ではアプリが対応していません |
| **セルを結合する** | CSV 書き出し時に正常に出力されません |

> ⚠️ **注意**: ヘッダー行のカラム名は英語・小文字・アンダースコア形式（例: `candidate_id`）で固定されています。絶対に変更しないでください。

### 1-3. ID ルール

- **既存行の ID は変更しない**（他のシートから参照されているため）
- **新しい行を追加するとき**は、既存 ID の末尾の番号を 1 増やす
  - 例: 候補者の最後が `10` なら次は `11`
  - 例: 応募の最後が `a12` なら次は `a13`

### 1-4. 日付フォーマット

- すべての日付は **`YYYY-MM-DD`（例: `2026-04-26`）** 形式で入力
- スプレッドシートが自動で日付形式に変換した場合は、セルの書式を「**テキスト**」に変更してから再入力

### 1-5. 真偽値（TRUE / FALSE）フォーマット

- `has_offer`（内定フラグ）、`opened`（開封済み）、`replied`（返信済み）は **`TRUE` または `FALSE`** の大文字で入力
- スプレッドシートのチェックボックスを使うと自動で `TRUE/FALSE` になるので推奨

## 2. スプレッドシート → CSV 書き出し手順

### 2-1. 1シートずつ書き出す方法

1. Google スプレッドシートを開く
2. 書き出したいシートのタブ（例: `candidates`）をクリック
3. メニューから **「ファイル」→「ダウンロード」→「カンマ区切り形式（.csv）」** を選択
4. ファイルがダウンロードフォルダに保存される
5. ファイル名を確認し、必要に応じて `candidates.csv` に変更する
6. 他のシートも同じ手順で書き出す

> 💡 **ポイント**: ダウンロードしたファイルのデフォルト名は「シート名.csv」になることが多いです。ファイル名が正確かどうかを必ず確認してください。

### 2-2. 書き出す必要があるファイル一覧

更新したシートの CSV のみ書き出せば OK です。更新していないシートは書き出し不要です。

| ファイル名 | シート名 | 主な更新タイミング |
|---|---|---|
| `candidates.csv` | candidates | 候補者の追加・ステータス変更 |
| `jobs.csv` | jobs | 求人の追加・ステータス変更 |
| `applications.csv` | applications | 応募の追加・ステータス変更 |
| `contacts.csv` | contacts | スカウト・DM 送信後 |
| `interviews.csv` | interviews | 面接実施後 |
| `evaluations.csv` | evaluations | 面接評価の入力後 |

### 2-3. 文字化けが発生した場合

CSV を Excel で開いた際に文字化けする場合は、Google スプレッドシートから直接ダウンロードした CSV（UTF-8）をそのまま使ってください。Excel を経由すると文字コードが変わる場合があります。

## 3. data フォルダへの配置手順

### 3-1. フォルダの場所

アプリのプロジェクトフォルダ内の `data` フォルダに配置します。

```
match-hire/           ← プロジェクトフォルダ
  └── data/           ← ここに CSV ファイルを置く
        ├── candidates.csv
        ├── jobs.csv
        ├── applications.csv
        ├── contacts.csv
        ├── interviews.csv
        └── evaluations.csv
```

### 3-2. 配置手順（Windows の場合）

1. エクスプローラーで `match-hire` フォルダを開く
2. `data` フォルダをダブルクリックして開く
3. ダウンロードした CSV ファイルをドラッグ＆ドロップする
4. 「ファイルを置き換えますか？」と聞かれたら **「置き換える」** をクリック

### 3-3. 配置手順（Mac の場合）

1. Finder で `match-hire` フォルダを開く
2. `data` フォルダをダブルクリックして開く
3. ダウンロードした CSV ファイルをドラッグ＆ドロップする
4. 「このファイルを置き換えますか？」と聞かれたら **「置き換える」** をクリック

### 3-4. 配置後のチェック

配置後、以下を確認してください。

- [ ] ファイル名が正確か（大文字・小文字・拡張子 `.csv` を含む）
- [ ] 更新したシートのファイルがすべて配置されているか
- [ ] ファイルサイズが 0 バイトになっていないか（書き出し失敗の可能性）

## 4. アプリ再起動手順

CSV ファイルを配置したあとは、アプリを再起動しないと変更が反映されません。

### 4-1. 開発環境（`npm run dev`）の場合

CSV ファイルを置き換えただけでは反映されません。以下の手順で再起動してください。

1. アプリを起動しているターミナル（黒い画面）を開く
2. `Ctrl + C`（Mac は `Control + C`）を押してアプリを停止する
3. 以下のコマンドを入力して Enter を押す

```
npm run dev
```

4. `ready - started server on 0.0.0.0:3000` のような表示が出たら起動完了
5. ブラウザで `http://localhost:3000/hire/dashboard` を開いて確認

### 4-2. 本番環境（Vercel など）の場合

本番環境では CSV ファイルをサーバーに配置する必要があります。担当エンジニアに連絡して再デプロイを依頼してください。

> 💡 **注意**: 本番環境では CSV ファイルはサーバー上に配置する必要があります。ローカルの `data` フォルダを更新しても本番環境には反映されません。

## 5. カラム変更禁止ルール（詳細）

### 5-1. 各シートの必須カラム一覧

以下のカラム名は変更・削除が禁止です。アプリが正しく動作するために必要な定義です。

#### candidates（候補者）

| カラム名 | 内容 | 入力例 |
|---|---|---|
| `id` | 候補者 ID（一意） | `1`, `2`, `11` |
| `name` | 氏名 | `山田 太郎` |
| `skills` | スキル（カンマ区切り） | `Java,Spring Boot` |
| `bio` | 経歴概要 | `大手SIerにて5年...` |
| `channel` | 接触経路 | `scout` / `dm` / `direct` |
| `status` | 選考ステータス | `書類選考` / `面接` / `内定` など |
| `valid` | 有効応募判定 | `有効` / `無効` / `未判定` |
| `updated_at` | 最終更新日 | `2026-04-26` |

#### applications（応募）

| カラム名 | 内容 | 入力例 |
|---|---|---|
| `id` | 応募 ID | `a1`, `a13` |
| `applied_at` | 応募日 | `2026-04-26` |
| `candidate_id` | 候補者 ID | `1` |
| `job_id` | 求人 ID | `j1` |
| `owner_id` | 担当者 ID | `o1` |
| `channel` | 応募経路 | `scout` / `dm` / `direct` |
| `validity` | 有効応募判定 | `有効` / `無効` / `未判定` |
| `invalid_reason` | 無効理由 | `年収条件ミスマッチ`（空欄も可） |
| `status` | 選考ステータス | `書類選考` / `面接` / `内定` など |
| `document_result` | 書類選考結果 | `通過` / `不通過` / `審査中` / `未実施` |
| `interview_stage` | 面接進行状況 | `2次面接`（空欄も可） |
| `has_offer` | 内定フラグ | `TRUE` / `FALSE` |

> その他シート（jobs / contacts / interviews / evaluations）のカラム定義は `docs/sheets_schema.md` を参照してください。

### 5-2. status の入力値（選考ステータス）

以下の8種類以外の値は入力しないでください。スペルミスや全角・半角の違いでデータが正しく表示されなくなります。

```
応募受付 / 書類選考 / 面接 / 内定 / 承諾 / 入社 / 不採用 / 辞退
```

## 6. バックアップ手順

データを失うリスクを減らすため、CSV を更新する**前**に必ずバックアップを取ってください。

### 6-1. スプレッドシートのバックアップ

Google スプレッドシートは自動で変更履歴を保存しています。

1. スプレッドシートを開く
2. メニュー「**ファイル**」→「**変更履歴**」→「**変更履歴を表示**」
3. 過去の状態に戻したい場合は、該当する日時をクリック →「**この版を復元**」

### 6-2. CSV ファイルのバックアップ

`data` フォルダに配置する前に、元のファイルをコピーして保管してください。

**推奨フォルダ構成**

```
match-hire/
  └── data/
        ├── candidates.csv          ← 現在使用中
        └── _backup/                ← バックアップフォルダ（手動で作成）
              └── 2026-04-26/       ← 日付フォルダ
                    ├── candidates.csv
                    ├── jobs.csv
                    └── （更新したファイルをコピー）
```

**手順**

1. `data` フォルダ内に `_backup` フォルダを作成（なければ）
2. `_backup` フォルダ内に今日の日付フォルダを作成（例: `2026-04-26`）
3. 現在の CSV ファイルをコピーして日付フォルダに貼り付け
4. 新しい CSV ファイルを `data` フォルダに上書き配置

> ⚠️ `_backup` フォルダはアプリには影響しません。ファイル名が `_` で始まる場合も同様です。

### 6-3. バックアップの保管期間

| 区分 | 保管期間の目安 |
|---|---|
| 日次バックアップ | 1ヶ月 |
| 月初バックアップ | 6ヶ月 |
| 重要な変更前のバックアップ | 無期限 |

## 7. トラブルシュート

### エラー一覧

---

#### ① アプリを開いたら「データの取得に失敗しました」と表示される

**原因と対処**

| 確認事項 | 対処 |
|---|---|
| `data` フォルダに CSV ファイルが存在するか | CSV ファイルを配置する（§3 参照） |
| ファイル名が正しいか（大文字・拡張子含む） | ファイル名を確認・修正する |
| `.env.local` の設定が `csv` になっているか | 担当エンジニアに確認を依頼 |
| アプリを再起動したか | ターミナルで `Ctrl+C` → `npm run dev` |

---

#### ② データは表示されるが、新しく追加した行が表示されない

**原因**: アプリが再起動されていない、または CSV が古い状態で配置されている。

**対処**:
1. CSV ファイルをもう一度書き出す（スプレッドシートを保存してから）
2. `data` フォルダに再配置する
3. アプリを再起動する（`Ctrl+C` → `npm run dev`）

---

#### ③ 名前や日付が文字化けして表示される

**原因**: CSV ファイルの文字コードが UTF-8 以外になっている（Excel 経由で保存した可能性）。

**対処**:
1. Excel を**経由せず**、Google スプレッドシートから直接ダウンロードしなおす
2. ダウンロードした CSV をそのまま `data` フォルダに配置する

> ⚠️ Excel で CSV を開いて保存すると、文字コードが「Shift-JIS」に変わり日本語が文字化けします。CSV の編集には必ず Google スプレッドシートを使用してください。

---

#### ④ 候補者一覧や応募管理のデータが空で表示される

**原因**: CSV の 1 行目（ヘッダー行）が変更または削除されている可能性があります。

**確認方法**:
1. CSV ファイルをメモ帳（Notepad）またはテキストエディタで開く
2. 1行目が以下のような英語カラム名になっているか確認する

```
id,name,skills,bio,channel,status,valid,updated_at
```

**対処**: ヘッダー行が変わっている場合は、`docs/sheets_schema.md` を参照して正しいカラム名に直す。または、バックアップから復元する。

---

#### ⑤ 一部の候補者・求人が表示されない

**原因**: `id` 列が空になっている行はアプリが自動でスキップします。

**確認方法**:
1. スプレッドシートで該当シートを開く
2. `id` 列に空欄がないか確認する

**対処**: `id` 列に正しい ID を入力して CSV を書き出し直す。

---

#### ⑥ 内定フラグや開封フラグが正しく反映されない

**原因**: `has_offer`・`opened`・`replied` の値が `TRUE`/`FALSE` 以外（例: `はい`・`○`・`1`）になっている。

**対処**:
1. 該当セルの値を `TRUE` または `FALSE`（大文字・英語）に変更する
2. スプレッドシートのチェックボックス機能を使うと自動で `TRUE/FALSE` になるため推奨

---

#### ⑦ 日付でフィルタすると正しい件数が表示されない

**原因**: 日付の書式が `YYYY-MM-DD` になっていない（例: `2026/4/26` や `26-Apr-2026`）。

**対処**:
1. スプレッドシートで日付カラムのセル書式を「**テキスト**」に変更する
2. `2026-04-26` 形式で再入力する
3. CSV を書き出し直す

---

#### ⑧ アプリがまったく起動しない（ターミナルにエラーが出る）

**対処**:
1. ターミナルのエラーメッセージをコピーする
2. 担当エンジニアに Slack / メールでエラーメッセージを共有する
3. その際、以下の情報も添えると対処が早くなります
   - いつから発生したか
   - 直前に行った操作（CSV の更新など）
   - エラーメッセージの全文

---

## よく使うチェックリスト

### データ更新時チェックリスト

```
□ スプレッドシートを上書き保存した
□ 更新したシートのタブを選択してから CSV を書き出した
□ ファイル名が正しい（例: candidates.csv）
□ 既存の CSV のバックアップを取った
□ data フォルダに上書き配置した
□ アプリを再起動した（Ctrl+C → npm run dev）
□ ブラウザで表示を確認した
```

### 毎月初めの確認チェックリスト

```
□ data フォルダの CSV が最新のスプレッドシートと一致しているか確認
□ _backup フォルダに月初スナップショットを保存
□ 1ヶ月以上前の日次バックアップを削除
```

## お問い合わせ

このガイドを読んでも解決しない場合は、担当エンジニアまでご連絡ください。  
その際、以下の情報を添えていただくと対応が早くなります。

1. 発生している症状（スクリーンショット添付推奨）
2. 直前に行った操作の手順
3. ターミナルに表示されているエラーメッセージ（あれば）
4. 更新した CSV ファイルの内容（個人情報は除く）
